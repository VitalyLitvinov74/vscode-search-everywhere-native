# Матрица пользовательских сценариев родного поиска VS Code

Дата среза: 2026-07-21.

## Решение

Все вычисление результатов выполняет VS Code. Расширение отвечает только за вход, выбор режима, передачу запроса, восстановление пользовательского контекста и совместимость.

Обозначения:

- `P0` — необходимо проверить до выбора интерфейса;
- `P1` — первая публичная версия;
- `P2` — следующий этап;
- `P3` — возможное развитие;
- `OUT` — не входит в объём работ.

## Вход и клавиатура

| Код | Сценарий | Способ | Приоритет | Проверяемый результат |
|---|---|---|---|---|
| `ENTRY-01` | Открыть поиск двойным Shift | Оркестрация | `P0` | Срабатывание стабильно и не мешает одиночному Shift |
| `ENTRY-02` | Назначить другое сочетание | Родная настройка VS Code | `P1` | Команда доступна в редакторе сочетаний |
| `ENTRY-03` | Не перезаписывать чужие сочетания молча | Оркестрация | `P1` | Конфликт виден, выбор остаётся за пользователем |
| `ENTRY-04` | Передать выделенный текст | Оркестрация | `P1` | Исходный запрос равен явному выделению |
| `ENTRY-05` | Передать слово под курсором | Оркестрация | `P2` | Поведение включается отдельно и не захватывает пробелы |
| `ENTRY-06` | Открыть последний режим | Оркестрация | `P2` | Настройка явно выбирает общий или последний режим |
| `ENTRY-07` | Работать только с клавиатуры | Родное поведение VS Code | `P1` | Ввод, выбор, просмотр, открытие и отмена не требуют мыши |

## Режимы

| Код | Сценарий | Источник результатов | Приоритет | Проверяемый результат |
|---|---|---|---|---|
| `MODE-01` | Найти файл | `Quick Open` | `P1` | Используется родное нечёткое ранжирование файлов |
| `MODE-02` | Найти файл и символ вместе | `Quick Open` с `search.quickOpen.includeSymbols` | `P0` | Расширение объясняет настройку и не меняет её без согласия |
| `MODE-03` | Найти символ рабочей области | префикс `#` | `P1` | Используется поставщик символов установленного языка |
| `MODE-04` | Быстро найти текст | `Quick Search`, префикс `%` | `P1` | Результаты и просмотр принадлежат VS Code |
| `MODE-05` | Найти действие | палитра команд, префикс `>` | `P1` | Используются родные названия и доступность команд |
| `MODE-06` | Выполнить сложный текстовый поиск | `workbench.action.findInFiles` | `P1` | Передаются строка и выбранные параметры |
| `MODE-07` | Перейти из быстрого текста в полную панель | родная кнопка `Quick Search` | `P1` | Запрос и результаты переходят в панель `Search` |
| `MODE-08` | Переключить категорию без знания префиксов | Оркестрация | `P0` | Пользователь выбирает понятное название режима |
| `MODE-09` | Переключить категорию клавишей `Tab` без потери строки | Оркестрация | `P0` | Возможность подтверждена на стабильном API или ограничение явно зафиксировано |
| `MODE-10` | Смешать файлы, символы, текст и действия в одном списке | Нет стабильного потребительского API текста | `OUT` | Не обещается до появления подходящего API VS Code |

| `TEXT-05` | Регулярное выражение | панель `Search` | `P1` | Проверку выражения и поиск выполняет VS Code |

## Совместимость и безопасность

| Код | Сценарий | Приоритет | Проверяемый результат |
|---|---|---|---|
| `COMPAT-01` | Минимальная поддерживаемая версия VS Code | `P0` | Все используемые команды существуют и принимают ожидаемые параметры |
| `COMPAT-02` | Windows, macOS и Linux | `P1` | Команды открывают одинаковые родные режимы |
| `COMPAT-03` | WSL, SSH и контейнер | `P1` | Поиск выполняется штатной стороной рабочей области |
| `COMPAT-04` | Несколько корней рабочей области | `P1` | Родные режимы соблюдают открытую рабочую область |
| `COMPAT-05` | Недоверенная рабочая область | `P1` | Расширение не запускает код проекта и не читает файлы самостоятельно |
| `COMPAT-06` | Отсутствующая языковая поддержка | `P1` | Режим символов объясняет пустую выдачу без ложной ошибки |
| `COMPAT-07` | Изменение внутренних команд VS Code | `P1` | Проверка совместимости выдаёт понятное сообщение и предлагает родную команду |
| `COMPAT-08` | Передача кода или запросов в сеть | `P1` | Сетевых вызовов и телеметрии содержимого нет |

## Технические проверки `P0`

| Код | Проверка | Решение считается подтверждённым, если |
|---|---|---|
| `SPIKE-PREFIX` | Передача префиксов и запроса в `workbench.action.quickOpen` | Файлы, `#`, `%` и `>` открываются с ожидаемой строкой |
| `SPIKE-FIND` | Передача параметров в `workbench.action.findInFiles` | Строка, маски, регистр, целое слово и регулярное выражение восстановлены в панели |
| `SPIKE-SHIFT` | Двойной Shift | Есть стабильное срабатывание, настраиваемая задержка и запасная команда |
| `SPIKE-TAB` | Переключение режима без потери запроса | Подтверждён доступный путь без чтения чужого родного поля ввода |
| `SPIKE-REMOTE` | WSL, SSH и контейнер | Диспетчер открывает те же родные поверхности без специального поискового кода |
| `SPIKE-VERSION` | Совместимость версий | Минимальная и текущая версии проходят одинаковый набор проверок |

## Граница первой публичной версии

В первую версию входят все строки `P1` и только подтверждённые строки `P0`.

Не входят:

- собственный индекс;
- `SQLite`, триграммы и отдельные базы;
- собственный `ripgrep` или другой внешний процесс;
- чтение и разбор файлов;
- собственный поставщик символов;
- собственное ранжирование;
- смешанный список текстовых совпадений с другими типами;
- предлагаемый API VS Code;
- вмешательство в DOM редактора;
- семантический или сетевой поиск.

## Измеримые критерии

- путь к нужному родному режиму требует не более одного вызова и одного выбора категории;
- исходный запрос не приходится вводить повторно до передачи в родную поверхность;
- расширение не создаёт файлов в проекте и не хранит содержимое исходного кода;
- нет фонового сканирования рабочей области;
- сочетания и настройки пользователя не меняются без явного действия;
- при недоступной команде пользователь получает рабочий запасной путь;
- сценарии проверены локально и хотя бы в одной удалённой среде.
