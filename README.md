# Indexed Code Search for VS Code

> Мгновенный локальный индексированный поиск по коду: файлы, символы и текст в едином окне.

Проект исследует, как перенести в VS Code сильные стороны `Search Everywhere` и `Find in Files` из PhpStorm, сохранив привычную экосистему VS Code.

Статус: исследование и проектирование. Реализация ещё не начата.

## Зачем

В больших рабочих областях повторное сканирование файлов на каждый запрос создаёт задержки, а поиск файлов, символов, команд и текста разбросан по разным интерфейсам. Цель проекта — поддерживать локальный постоянный индекс и давать предсказуемую выдачу уже с первых введённых символов.

## Целевой пользовательский опыт

- единое окно для файлов, символов, текста и команд;
- мгновенный поиск подстроки по постоянному индексу;
- живой просмотр результата и навигация только с клавиатуры;
- области поиска, маски файлов, исключения, регистр, целые слова и регулярные выражения;
- постепенная индексация без блокировки поиска;
- точечное обновление индекса после изменений файлов;
- восстановление повреждённого индекса без ручной очистки каталогов;
- работа локально, без передачи исходного кода и поисковых запросов в сеть;
- поддержка локальных проектов, WSL, SSH и контейнеров как обязательное архитектурное требование.

## Принципы

1. Сначала измерения: скорость первой индексации, задержка запроса, размер индекса, память и нагрузка на процесс расширений.
2. Индекс хранится в частном хранилище расширения, а не внутри проекта.
3. Пока индекс строится, поиск продолжает работать по уже готовой части и через резервный механизм.
4. Поиск файлов, полнотекстовый поиск и символы имеют разное происхождение данных, но единое ранжирование и интерфейс.
5. Формат индекса версионируется, проверяется и допускает безопасную пересборку.
6. Архитектура не должна зависеть от нестабильного API VS Code, запрещённого для публикации в Marketplace.

## Текущий ориентир

Наиболее вероятное направление первой технической проверки — гибридный движок:

- постоянный индекс имён файлов и содержимого;
- триграммный отбор кандидатов для быстрого поиска подстрок и регулярных выражений;
- точная проверка совпадений перед показом;
- символы через стабильные поставщики языка VS Code;
- наблюдатели файлов плюс периодическая сверка состояния;
- отдельное наложение для несохранённых документов.

Выбор между `SQLite FTS5` через WebAssembly и специализированным триграммным форматом будет сделан по результатам воспроизводимых испытаний.
